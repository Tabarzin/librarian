# Автоматизированная система сбора, обработки, каталогизации материалов в базе данных и поиска по базе.

## Добавление материалов в базу
- n8n-воркфлоу
- База данных: Airtable
- Хранилище файлов: Google Drive
  
### Функционал системы
Система автоматически обрабатывает входящие материалы из Telegram и добавляет их в базу знаний. Воркфлоу выполняет следующие операции:​
- Прием сообщений через Telegram Trigger с поддержкой текстовых постов, URL-ссылок, PDF-документов и DOCX-файлов
- Извлечение метаданных: автор поста, комментарий отправителя, исходный текст, ID группы, тип контента
- Парсинг и извлечение URL из текста сообщений и подписей (основная ссылка + дополнительные ссылки)
- Определение источника материала (канал Telegram, пересланное сообщение или веб-ссылка)

### Обработка контента
Для обработки различных типов данных используются два специализированных субворкфлоу:
- SUB - URL/Text Processor обрабатывает веб-ссылки и текстовые посты:​
  - Загрузка содержимого статей через Jina AI Reader API (https://r.jina.ai)
  - Парсинг контента: извлечение заголовка, даты публикации, PDF-ссылок
  - Предварительная очистка текста от служебной информации, навигационных элементов и метаданных Jina
  - Форматирование дат в формат MM/DD/YYYY
  - Обработка ошибок загрузки с уведомлениями

- SUB - File Processor обрабатывает документы:
  - Получение файлов из Telegram Bot API по file_id
  - Маршрутизация по типу: PDF или DOCX
  - Загрузка файлов в Google Drive
  - Извлечение текста из PDF с сохранением метаданных
  - Формирование ссылки webViewLink для доступа к файлу
  - Сохранение метаданных Telegram (sender, postAuthor, senderComment, originalPostText)

## Обработка через AI
Подготовленные данные передаются в AI Agent (Google Gemini) для структурированного анализа:
- ​Извлечение или перевод названия статьи/исследования на русский язык
- Генерация краткого резюме (summary) на русском языке (2-4 предложения)
- Автоматическое создание тегов
- Определение источника публикации (журнал, сборник, конференция или веб-домен)
- Валидация и нормализация даты публикации
- Обработка усеченного текста для PDF (лимит 15000 символов для AI)

## Обработка ошибок
Реализована система классификации и уведомлений об ошибках:
​
- Обработка rate limit (HTTP 429) с возможностью повторной попытки
- Детекция временной недоступности Airtable (HTTP 503)
- Валидация прав доступа (HTTP 401/403)
- Обработка ошибок AI Agent с отправкой уведомлений в Telegram
- Уведомления о неподдерживаемых форматах файлов
- Уведомления об успешном добавлении материала с данными из Airtable

## Библиотекарь
Воркфлоу: Librarian

Реализован интеллектуальный Telegram-бот для семантического поиска по базе знаний Airtable с использованием AI-ассистента на базе Google Gemini. Библиотекарь обеспечивает интеллектуальный поиск и аналитику данных из базы знаний через диалоговый интерфейс Telegram. Система автоматически выбирает релевантные записи из Airtable, форматирует их для обработки AI и генерирует структурированные ответы на естественном языке.
